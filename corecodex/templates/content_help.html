» General CoreCodex Questions

Is CoreCodex free?

Yes, this website is 100% free! In the future we will open up a service for professor / academic level use which may or may not require a subscription service fee. For end-users, including students, this site will always remain free.

How was CoreCodex developed?

CoreCodex was written entirely in the Python programming language using the Pylons framework. We are currently hosted on DreamHost, a great website hosting company!

Who developed CoreCodex?

Jeremy Bridon, under his company Core S2 - Software Solutions. Feel free to contact us if you have any questions for the developer.

There's something broken! How do I submit a bug?

CoreCodex is deep in alpha testing; we expect many things to break and many of our programming challenges to possibly be reporting false-positives for judging results. Regardless, if you want to report a bug, feel free to contact us directly. There is also a bug-reporting system that you can access using the Guest/guest username/password found here, on our Mantis bug-tracking system.

Why don't I see the text input I'm programming against?

When submitting your code, we never show the output results nor the test inputs. The reason is to keep things fair and to prevent you from writing a dummy program that gives you the expected outputs so that you can trick the system into thinking you actually wrote a solution.

What languages do you support?

In this alpha-state of testing, we only support Python. In the near future when the application matures enough to be labled "beta", we will support C, C++, and Java. In the future we might also support even more languages, such as Lisp, C#, and even Visual Basic.

How do you render those charts in challenge descriptions?

We use Google Charts, which is a neat service that allows us to render these images off-server.

» Python Programming Questions

What version(s) of Python do you support?

We currently support Python2.4, though soon plan to support Python 2.7 (latest V2 release) and Python 3.1!

Should I use tabs or four-spaces?

Python is very format-sensitive. When writing a block of code, the amount of space you have representing a block of code must be consistent: either "tab" with four spaces or tab with the actual ascii-tab value (using your keyboard's tab key). CoreCodex and CoreJudge assume you are tabbing using the ascii-tab values, and thus all code submitted should use the tab key from your keyboard. We are aware this is not what some users prefer, and thus we are working on creating a preference for this.

What libraries are included?

The only imports / libraries available to Python developers are the math, random, and regex libraries. This is for security reasons.
