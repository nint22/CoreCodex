Introduction

CoreCodex is an academic tool used for autmated testing of a user's code. Student and professor alike can use CoreCodex, as well as the general public to help with learning a new language, learning new features, or developing better code techniques.

CoreCodex contains many programming problems, as well as alows the creation of new programming challenges, so that students and professors can quickly test themselves or others. These challenges aren't just about testing code, but are also meant as a way to learn the real-world implications of data structures and algorithm design.

Similar to the online ACM judge and ACM collegiate competative programming events, this system accepts multiple languages and types of problems. Currently supported languages include Python, C, C++, and Java.

CoreCodex was developed by Jeremy Bridon of CoreS2 Software Solutions. Check out his blog and keep yourself updated! The site is currently in an alpha state and is thus heavily under development. Send him an email if you see something broken.
